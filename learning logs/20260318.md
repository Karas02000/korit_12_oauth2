# OAuth2 구현
1. OAuth2란?
    - Open Authorization 2.0의 축약어로 사용자의 비밀번호를 직접 받지 않고 구글/ 카카오같은 외부 서비스를 토애 사용자를 인증(Authentication)하는 표준 방법

    - 예시로 GitHub 로그인을 할때의 구글로 로그인하기와 같은 것을 생각해볼 수 있다.

    - 하지만 sign up할 때에는 결국 username과 email을 쓰고 비밀번호까지 입력하기로 했다. 그렇다면 이미 회원정보가 웹 사이트의 DB 내에 존재하는데, 로그인은 구글로 한다고도 볼 수 있다.

2. 왜 필요한가?
    - 기존 방식의 경우 사용자가 사이트에 username / password를 입력하게 되면 서버가 구글 대신에 로그인을 해준다. -> 사용자의 비밀번호가 서버에 노출되어버린다.

    - OAuth2 방식의 경우, 구글로그인 페이지에서 구글에 로그인을 시도하고 -> 서버는 _허가 코드_를 받게된다. -> 그리고 이 허가코드를 통해서 사용자 정보를 받아오게 되기 때문에 구글만 구글의 비밀번호를 확인할 수 있다.

- 이상에서 중요한 점은,
    1. 사용자는 구글 페이지에서 직접 로그인한다.
    2. 구글은 Authorization Code를 주고, 이를 통해서 Access Token을 받게 된다.
    3. Access Token으로 구글 API에서 사용자의 정보를 받아온다.
    4. 웹 사이트의 서버는 해당 사용자의 정보와 일치하는 것을 발견한 후, 웹 사이트의 JWT를 발급해서 프론트로 전달한다.
